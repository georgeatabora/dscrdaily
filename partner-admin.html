Airtable — Partners Table Schema
Add these fields to your Airtable base alongside your existing Deals and Subscribers tables. Create a new table called Partners and another called ReferralEvents.
Partners Table
Name → Single line text (partner full name) Code → Single line text (unique e.g. JONES-REALTY) PIN → Single line text (4-digit access PIN) Email → Email (notification email) Phone → Phone (optional SMS alerts) Role → Single select → Real Estate Agent → Builder / Contractor → Investor → Other Company → Single line text Active → Checkbox (default: checked) NotifSignup → Checkbox (email on signup) NotifInquiry → Checkbox (email on inquiry) NotifWeekly → Checkbox (weekly summary) Notes → Long text (internal only) Created → Date (auto)
ReferralEvents Table
event → Single select → page_visit → signup → inquiry ref_code → Single line text (partner code) partner → Single line text (partner name) lead_name → Single line text lead_email → Email lead_role → Single line text lead_type → Single select (subscriber/lead/inquiry) page → Single line text timestamp → Date + time user_agent → Single line text
Make.com — Partner Notification Workflow
This scenario runs every time your site sends a referral event to your webhook. It looks up the partner, checks their notification prefs, and sends them an email.
1Trigger: Webhook
Create a new webhook in Make.com → copy the URL → paste it into the REFERRAL.WEBHOOK constant in your landing page script
2Module: Airtable — Search Records
Table: Partners · Filter: {Code} = "{{1.ref_code}}"
This finds the partner record so you can get their email and notification prefs
3Module: Router (branch by event type)
Branch A: {{1.event}} = "signup" AND {{2.NotifSignup}} = true
Branch B: {{1.event}} = "inquiry" AND {{2.NotifInquiry}} = true
4Module: Email (Resend or Gmail)
To: {{2.Email}}
Subject: "Someone from your network just connected with DSCR Daily"
Body: Use the email template below
5Module: Airtable — Create Record
Table: ReferralEvents · Map all fields from {{1.*}}
This logs every event permanently in your database
6Module: Airtable — Update Record
Table: Partners · Increment the click/signup/inquiry counters on the partner record

Landing Page Integration — Already Done ✓
The referral tracking code is already embedded in your index.html (dscr-daily-landing.html). The only values you need to replace are the three constants at the top of the REFERRAL object. Once your Make.com webhook URL and Airtable credentials are in, the system is fully live.
// In index.html — find and replace these three values: const REFERRAL = { WEBHOOK: 'https://hook.us1.make.com/YOUR_WEBHOOK_ID', AIRTABLE_KEY: 'YOUR_AIRTABLE_API_KEY', AIRTABLE_BASE: 'YOUR_BASE_ID', ... } // Partner links look like: // dscrdaily.com?ref=JONES-REALTY // dscrdaily.com?ref=SMITH-BUILD // dscrdaily.com?ref=DEREK-T // Partner portal is at: // dscrdaily.com/partner-portal.html
